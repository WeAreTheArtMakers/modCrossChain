LI.FI Aggregator + Wallet-Only Execution

Bridge assets across chains without building a bridge protocol.

modCrossChain is a clean, dark, non-custodial bridge interface for Ethereum, BNB Chain, and Polygon. It requests the best route from LI.FI, sends transactions through the user's wallet, and keeps private keys entirely client-side.

Interface

A compact bridge card with route clarity built in.

The product focuses on the decision surface that matters: chain choice, token lookup, amount entry, route cost, ETA, and transaction follow-through. The dark visual system keeps attention on execution rather than decoration.

Product Highlights

Minimal surface, practical routing, clear execution.

The interface is designed to feel fast and trustworthy: source and destination chain selectors, searchable token metadata, debounced route requests, route cost visibility, and a transaction status modal that follows the LI.FI execution lifecycle.

Route intelligence

Fetches cheapest or fastest routes with LI.FI and shows gas cost, bridge fee, receive amount, and ETA.

Wallet-native flow

All signing happens in the connected wallet. No custom custody, no private key handling, no backend signer.

Deployment-ready

Prepared for Railway deployment, GitHub publishing, WalletConnect configuration, and integrator fee monetization.

How It Works

Aggregator-first, wallet-native, easier to maintain.

01

Connect a wallet

MetaMask or WalletConnect establishes the client-side signer and detects the active chain.

02

Request the best route

LI.FI resolves the cheapest or fastest route for the selected token, amount, and destination.

03

Execute and track

The route is executed in the wallet, with status updates, transaction hash visibility, and explorer links.

Stack

Built with a modern frontend and aggregator-first architecture.

The bridge UI is intentionally lightweight: App Router for structure, wagmi + viem for wallet actions, and LI.FI for cross-chain route discovery and execution.

Security

Non-custodial by design

  • Client-side execution only.
  • Private keys never leave the wallet.
  • Token address validation and zero-amount rejection before route requests.
  • Optional LI.FI integrator fee support without changing custody model.
